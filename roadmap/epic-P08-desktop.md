# Epic: 桌面应用

- **ID:** E-P08
- **状态:** 进行中
- **目标:** 在已具备 Tauri 桌面端能力的基础上，完成 **桌面 App（Tauri）与 PC Web 显示效果一致性收口**，让用户在桌面端看到的是同一套产品体验，而不是“功能一样、观感像两个版本”。

## 当前基线

- 已完成：Tauri 桌面版 `v0.1.0 ~ v0.1.2`（Windows + Linux）
- 当前主线：桌面 App 与 PC Web 的视觉与布局一致性
- 技术策略：继续使用**现有前端栈 + Tauri 壳层**完成本轮收口，**不引入 Pixi.js**
- 风险待核实：`Windows 双击无响应` 暂不作为当前主线问题，只在最终发版回归中顺手核验

## 说明

- **Epic 只负责定义目标、范围、验收标准与推进顺序**。
- **正式执行载体统一为 GitHub Issues**。
- 本文件**不再使用 `Txx` / `Task` 编号体系**，避免和旧流程混淆。

## 当前下一步

- 先创建 GitHub Issue：**基线冻结——PC Web vs Tauri 对比口径统一**
- 本步目标：先把 `PC Web / Tauri Desktop` 的对比口径固定下来，避免后续“一致性”没有统一标尺
- 本步最小产物：固定视口、固定主题、固定语言、固定测试数据，以及 5 个目标页面的 Web / Tauri 成对基线截图
- 本步完成后，再进入“全局壳层一致性修正”

## 本轮范围（P0）

### 1. 全局壳层一致性

- 窗口内边距与页面容器宽度
- 顶部 / 底部安全边距
- 高度链路（首屏是否被拉满、压扁、贴边）
- 滚动行为（是否出现多余滚动条 / 首屏裁切）
- 字体、字号、间距、圆角、阴影等基础视觉参数

### 2. 关键页面一致性

- 番茄钟首页
- 农场页
- 仓库页
- 设置页
- 个人资料页

### 3. 验收口径固定化

- 对比对象：`PC Web` vs `Tauri Desktop`
- 固定视口：`1440x900`、`1280x720`
- 固定主题：`Light`、`Dark`
- 固定语言：`简体中文`
- 固定测试数据：使用统一账号 / 统一测试状态，避免数据差异干扰验收

## 不在本轮范围

- 移动端专项适配
- 社交与分享功能
- 商业化功能
- 新功能扩展
- Pixi.js / 新渲染路线
- Windows 启动问题专项排查（仅作为发版回归项）

## 验收标准（本轮）

- [ ] 已建立固定对比口径，能稳定产出 `Web / Tauri` 成对截图
- [ ] 在 `1440x900` 与 `1280x720` 下，关键页面首屏无新增横向滚动条
- [ ] 在 `1440x900` 与 `1280x720` 下，关键页面首屏无明显裁切、贴边、塌陷、错位
- [ ] 番茄钟首页的主计时器、按钮组、顶部信息区在 Web / Tauri 中观感一致
- [ ] 农场页的九宫格位置、上下留白、主视觉构图在 Web / Tauri 中观感一致
- [ ] 仓库 / 设置 / 个人资料页无明显字体漂移、换行异常、控件错位
- [ ] `Light / Dark` 两套主题均完成回归检查
- [ ] 最终发版产物齐全：`version + commit + desktop build + compare artifacts`
- [ ] Windows 安装包双击可正常启动（仅作最终回归检查项）
